# Security Policy

## Responsible disclosure

Please report security vulnerabilities privately through the designated security contact/channel for this repository.

Do not disclose vulnerabilities publicly until coordinated remediation is complete.

## Payment abuse/fraud incident categories

- account takeover and unauthorized payout attempts
- synthetic identity or mule-account abuse indicators
- transaction laundering patterns
- policy/risk control bypass attempts
- replay or idempotency abuse
